Proceso de arranque de Android: desde Boot ROM hasta Zygote

Recorrido paso a paso por la cadena de arranque: Boot ROM, Bootloader, carga del kernel, primer y segundo estadio de init, servicios nativos, Zygote y arranque del System Server.

Cadena resumida

  1. Boot ROM ejecuta código inicial del SoC.
  2. Bootloader inicializa memoria, verifica imágenes y elige slot.
  3. Kernel Linux arranca con ramdisk / init_boot / vendor_boot según versión.
  4. init monta particiones, carga SELinux y ejecuta scripts .rc.
  5. Zygote prepara ART y se convierte en raíz de procesos Java/Kotlin.

La secuencia exacta depende de versión Android, SoC, particiones, AVB, A/B y configuración del fabricante.

Lectura forense

Qué mirar sin inventar conclusiones

Bootloader / AVB

Estado de bloqueo, verified boot state, particiones vbmeta/boot/vendor_boot/init_boot y coherencia con el modelo/versiones.

Kernel / ramdisk

Imagen de arranque, módulos tempranos, parámetros androidboot, ramdisk genérico/vendor y fstab.

Init / persistencia

Scripts init*.rc, servicios añadidos, clases, sockets, propiedades persistentes y binarios ejecutados por init.

Zygote / apps

Relación con procesos Java, System Server, app_process, clases de arranque y namespaces; no atribuir malware solo por presencia de Zygote.

En análisis pericial: documentar versión, fuente del artefacto, hash, método de extracción, limitaciones y trazabilidad. No extrapolar un flujo AOSP genérico a un terminal concreto sin contrastar firmware, fabricante y particiones reales.
